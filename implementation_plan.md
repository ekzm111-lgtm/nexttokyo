# 미구현 기능 추가 및 버그 수정 구현 계획

현재 프로젝트의 서버([server.js](file:///d:/01-%EC%9E%91%EC%97%85/%EC%83%88%20%ED%8F%B4%EB%8D%94%20%282%29/server.js))와 관리자 페이지([admin.html](file:///d:/01-%EC%9E%91%EC%97%85/%EC%83%88%20%ED%8F%B4%EB%8D%94%20%282%29/admin.html))를 분석한 결과, 데이터 생성은 가능하나 기존 데이터의 삭제 및 상세 관리가 불가능하며, 정적 파일 경로 설정에 오류가 있어 이미지가 제대로 표시되지 않는 문제가 있습니다. 이를 해결하기 위한 계획입니다.

## Proposed Changes

### [Server] server.js
- **정적 경로 수정**: `app.use(express.static(path.join(__dirname, '.')))`가 루트의 [index.html](file:///d:/01-%EC%9E%91%EC%97%85/%EC%83%88%20%ED%8F%B4%EB%8D%94%20%282%29/index.html)을 기본으로 잡고 있어, API 호출 시 HTML이 반환되는 문제를 방지하기 위해 정적 파일 제공 순서를 조정하거나 명확히 분리합니다.
- **삭제 API 추가**:
    - `DELETE /api/admin/news/:id`
    - `DELETE /api/admin/gallery/:id`
    - `DELETE /api/admin/contacts/:id`
- **상세 데이터 조회 API 보완**: 각 테이블의 개별 항목 조회 엔드포인트 추가.
- **연락처(Inquiry) 테이블 날짜 형식 통일**: `CURRENT_TIMESTAMP` 대신 클라이언트와 일치하는 형식(YYYY.MM.DD) 또는 호환 가능한 형식으로 저장되도록 수정.

### [Admin] admin.html
- **삭제 버튼 추가**: 공지사항, 시공 실적, 문의 내역 리스트에 삭제 버튼을 추가하고 API와 연동합니다.
- **리스트 표시 로직 개선**: 테이블 레이아웃을 다듬고 데이터가 없을 때의 처리를 강화합니다.
- **이미지 업로드 경로 수정**: 업로드 완료 후 반환되는 URL이 클라이언트에서 즉시 미리보기 가능하도록 수정합니다. (현재 `./uploads` 경로와 서버 설정 간의 불일치 해결)

### [Main] index.html
- **문의 폼 연동**: 현재 폼의 ID(`contactForm`, `cName` 등)와 스크립트의 [submitInquiry](file:///d:/01-%EC%9E%91%EC%97%85/%EC%83%88%20%ED%8F%B4%EB%8D%94%20%282%29/index.html#501-522) 함수에서 사용하는 ID가 일치하지 않는 부분을 수정하여 실제 데이터가 서버로 전송되도록 합니다.
- **시공 실적 렌더링**: 갤러리 이미지 경로가 `/uploads/`로 시작하는 경우와 로컬 이미지(`./img_...`)인 경우를 모두 처리할 수 있도록 렌더링 로직을 보강합니다.

## Verification Plan

### Automated Tests
- `curl` 또는 `Postman` 기능을 대체하는 테스트 스크립트(tmp/test-api.js)를 작성하여 각 엔드포인트(GET, POST, DELETE)가 정상 작동하는지 확인합니다.
    - `node tmp/test-api.js` 실행

### Manual Verification
1. **관리자 페이지**:
    - 새로운 공지사항과 갤러리 항목을 등록하고 리스트에 나타나는지 확인.
    - 등록된 항목을 삭제했을 때 리스트와 DB에서 사라지는지 확인.
    - 이미지 업로드 후 실적 등록 시 이미지가 엑박 없이 표시되는지 확인.
2. **메인 페이지**:
    - 문의 폼을 작성하고 제출 후 "성공" 알럿이 뜨는지 확인.
    - 관리자 페이지의 "문의 내역 관리"에서 방금 제출한 내용이 보이는지 확인.
    - 최신 공지사항과 갤러리 데이터가 인덱스 페이지에 반영되는지 확인.
